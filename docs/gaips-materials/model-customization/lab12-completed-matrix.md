# Lab 12 Completed Customization Matrix

| Approach | Primary Security Goal | Data/Artifact Needed | Main Risk | Required Control | Evaluation Evidence | When To Avoid |
| --- | --- | --- | --- | --- | --- | --- |
| Prompt engineering | Improve behavior without changing weights | System prompt, prompt templates, test prompts | Prompt injection, brittle instructions, hidden policy drift | Version prompts, separate instructions from retrieved text, regression tests | Promptfoo or Inspect AI prompt regression | When behavior requires domain knowledge not present in prompt/context |
| RAG | Ground answers in approved sources | Sanitized corpus, metadata, vector index | Poisoned documents, retrieval leakage, missing citations | Document review, metadata filters, citation enforcement, conflict detection | RAG poisoning results and citation correctness table | When source data cannot be trusted or access control cannot be enforced |
| Guardrails | Detect/block unsafe prompts, chunks, tool output, responses | Policy labels, classifiers, deterministic gates | False positives/negatives, overreliance on model classifiers | Layer deterministic checks with Prompt Guard/Llama Guard/Model Armor review | Guardrail regression summary | When guardrails are treated as the only authorization boundary |
| Prompt Guard | Local jailbreak/injection signal | Approved model path or fixture output | Context window limit, threshold tuning, label ambiguity | Chunk inputs, tune thresholds, treat third-party content strictly | `guardrails/prompt-guard-results.json` | When model access/licensing is not approved |
| Model Armor | Managed prompt, document, response, DLP screening | GCP project/template or fixture output | Cloud dependency, IAM scope, latency, inspect/block rollout risk | Inspect-only rollout, least-privilege IAM, DLP/redaction decision table | `guardrails/model-armor-results.json` | When data cannot leave approved environment |
| Llama Guard 3 | Safety taxonomy classification | Local/hosted classifier or fixture output | Taxonomy mismatch with app-specific policy | Map hazard categories to app policy and deterministic decisions | `guardrails/llama-guard-3-results.json` | When hazard classes do not cover the app risk |
| Fine-tuning | Teach durable domain behavior or format | Curated training/eval dataset, base model, training script | Data leakage, memorization, regressions, cost | Dataset review, held-out safety evals, model signing, rollback | Safety regression and model artifact evidence | When RAG or prompting solves the need with lower risk |
| LoRA/adapters | Efficient targeted customization | Adapter weights, base model, training config | Adapter provenance, unsafe merge, hidden behavior changes | Track base model digest, adapter source, eval before merge | Adapter notes plus before/after evals | When adapter source cannot be verified |
| Moderation | Enforce content policy at input/output boundaries | Policy taxonomy, test set, moderation provider or local classifier | Policy mismatch, missing context, sensitive logging | Pre/post moderation, redaction, appeals/human-review workflow | Moderation regression cases and false-positive review | When moderation output is used as sole access-control decision |
